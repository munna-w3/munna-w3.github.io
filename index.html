MetricOmmPool bin-ladder simulator

Define a ladder of bins around a mid price, then trade across them. Compares three models side by side: old (the original buggy SwapMath — leaks value to traders), current (the patched SwapMath now in metric-core — pool never loses, may over-charge), and fixed (the ETH-flat-anchor — conserves exactly). The swap math runs in BigInt / Q64.64 exactly like the Solidity contract (float is used only for drawing). token0 = ETH, token1 = USDC. Cursor 0 = equilibrium (pool spot = mid).

oracle (bid / ask)

ladder

Bins above mid seed as ETH, below as USDC. Widths compound like mid·(1+w). Total swap fee = oracle spread + the bin's additional fee. Set bid = ask for a zero base fee (isolates the leak).

trade

Buy = trader takes ETH, pays USDC, cursor moves up. Sell = the reverse. Both models execute the same trade so you can watch them diverge.

re-anchor & arbitrage

Re-anchor scales every bin's price range by newMid/oldMid (bins are distances from mid). "Arb to mid" trades the pool back to spot = mid; "Round-trip test" runs a net-zero-ETH oscillation and checks value conservation.

drain scenario — step-through leak demo

One bin, 100,000 USDC, zero fee. Sell ETH in, oscillate, then drain all ETH back out — net ETH ends at 0, so a value-conserving pool must return ≥ 100,000 USDC. Steps run the real integer engine for all three models at once: old ends below 100,000 (leaked to the trader), current ends above (over-charged, pool safe), fixed lands on 100,000 (conserved).
